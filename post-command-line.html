Command Line

As for me, the command line is a part of my developer and not developer life. I create new files and copy olds one in a console, I commit my changes in a console and open some programs from it. And I think if you know some simples commands you can make your life a little bit easier.

> sudo. It's the first part of the command, for example, 'sudo mkdir new_directory', which gives you superuser rights. In the example, you can create directory everywhere you want, but if you decide to change directory rights or rename directory, you will need to use 'sudo' command too. You should be careful with this command because it has many opportunities but a lot of dependencies. Every time before using this command think maybe you don't need to use it.

> cd. This command is used for directory surfing. If I want to change my current directory 'lakie' to another directory 'developer', which is in another directory I just write to console 'cd ../another_dir/ developer'. Too easy? What do you think?

> ls -la. I use this command for looking in directory what files and directories it is inside. -la are flags. -l for lists all contents of a directory in long format and -a for lists all contents in the working directory, including hidden files and directories.

> pwd. A simple command for looking a where you in the directory path.

> cp. This command copy a directory (with flag -r) or file to another directory. For example, cp iphone7.log ../devices/, this command copy file iphone7.log from the current folder to another 'devices' directory.

>mkdir. Command to create a new directory.

>touch. Command to create a new file. I like this command because it helps me to create files what I need faster than programs interface.

>cat. If you write cat config.log, you will see all file's content in the terminal.

Bonus. > brew. As a Mac user, I use brew command for installing necessary tools.
